# Release Notes: v1.0 - Production Launch

**Release Date**: 2025-01-27  
**Version**: 1.0.0  
**Status**: Production Ready

## Overview

SIGN X Studio Clone v1.0 is the first production release, providing a complete sign design and engineering platform with deterministic calculations, full traceability, and automated report generation.

## New Features

### Core Functionality

- **8-Stage Workflow**: Complete project lifecycle from site analysis to submission
- **Interactive Canvas**: Real-time cabinet design with automatic geometry calculation
- **Automatic Pole Selection**: Intelligent filtering based on loads and constraints
- **Foundation Design**: Direct burial and baseplate design paths
- **PDF Reports**: Automated 4-page engineering reports
- **Confidence Scoring**: Quantified uncertainty (0.0-1.0) for all calculations

### API Features

- **35+ Endpoints**: Complete REST API
- **Envelope Pattern**: Standardized response format
- **Deterministic Outputs**: Content SHA256 for verification
- **Idempotency**: Safe retry with Idempotency-Key
- **Optimistic Locking**: ETag-based concurrency control

### Integration Features

- **KeyedIn CRM**: Webhook integration
- **OpenProject**: Ticket dispatch
- **Email Notifications**: Automated alerts
- **MinIO**: Object storage for files

## Known Issues

### Limitations

1. **Engineering Review**: Depths >5 feet require manual review (by design)
2. **Pole Options**: Limited to AISC catalog (200+ options)
3. **Report Caching**: Reports cached for 24 hours (deterministic)

### Workarounds

- **Low Confidence**: Review assumptions, adjust inputs, or request engineering review
- **No Pole Options**: Reduce loads, change material, or increase height
- **Slow Response**: Check internet connection, clear cache, or contact support

## Upgrade Notes

### From Beta (v0.x)

**Breaking Changes:**
- API responses now use Envelope format
- Some endpoint paths changed
- Authentication now required for all endpoints

**Migration:**
- Update API client to parse Envelope
- Use new endpoint paths
- Include authentication headers

See [Migration Guide](../migration/v2-to-v3.md) for details.

### Database Migrations

Run migrations before deployment:
```bash
cd services/api
alembic upgrade head
```

### Environment Variables

New required variables:
- `APEX_ENABLE_ENVELOPE=true`
- `APEX_CONTENT_SHA_ENABLED=true`
